# Dark Application Dashboard Design

## Goal

将 TaskHub 整套前端统一为深色工作台，并把“协同总览”升级为高密度运营仪表盘。参考 Sub2API 仪表盘的信息层级与视觉节奏，但只展示当前 TaskHub 接口能够真实计算的数据。

## Scope

- 全局启用 Ant Design 深色算法，统一导航、工具栏、表单、表格、弹窗、流程画布和详情面板的深色语义。
- 保留现有页面结构、导航和业务交互，不重构 API 或状态管理。
- 总览新增时间范围筛选、八项真实指标、状态分布、七日趋势、风险任务、最近任务和最近事件。
- 进入总览时刷新任务、人工待办和 Agent 数据，避免回到总览后显示旧状态。
- 不展示余额、Token、费用、响应时间或 Agent 在线率等当前接口没有的数据。

## Visual Direction

- 背景使用深蓝黑，面板使用略亮的深灰蓝，边框保持低对比。
- 青色用于主要交互，绿色表示成功，琥珀色表示待处理，玫红色表示异常，紫色只作次级区分。
- 卡片圆角不超过 8px，不增加装饰光球或大面积发光效果。
- 桌面端指标为四列两行；平板两列；移动端单列，所有长文本必须省略或换行，不得横向溢出。

## Data Model

- `total`: 筛选范围内任务总数。
- `today`: 本地自然日创建的任务数。
- `running`: `running`。
- `succeeded`: `succeeded`。
- `risk`: `failed + blocked + partial`，取消任务不计入风险。
- `humanPending`: 当前接口返回的人工待办数量。
- `completionRate`: `succeeded / total`，空数据为 `0`。
- `agents`: 当前可见 Agent 数量，不推断在线状态。
- 七日趋势按 `created_at` 的本地日期聚合；无效日期忽略。

## Interaction

- 时间范围支持“全部 / 今日 / 近 7 天”，只影响总览统计、趋势、风险和最近任务；最近事件始终按事件时间展示最新记录。
- “发布任务”继续进入现有发布页。
- 风险任务可进入现有任务列表继续处理；最近任务复用现有紧凑表格，不增加新后端接口或跨页详情状态。
- 全局顶部刷新保留；进入总览时自动刷新相关数据。

## Verification

- 纯函数测试覆盖状态口径、时间筛选、完成率、风险排序、七日趋势和空数据。
- 刷新策略测试覆盖管理员和普通用户进入总览时的数据目标。
- 样式契约测试覆盖 Ant Design 深色算法、全局深色变量和总览响应式栅格。
- 运行前端全量测试与生产构建，并检查 1440×1000、1024×768、390×844 三档截图。
