# 任务详情页布局优化设计

## 目标

在不修改任务接口、数据结构和执行逻辑的前提下，优化任务详情弹窗的信息层级、图标表达和交互反馈，并消除自动规划执行轮次区域的内容裁切与嵌套滚动遮挡。

## 方案选择

采用现有单页纵向详情结构的小范围重构。

- 不新增页签，避免把上下文、结果和执行轮次拆散。
- 不引入新的 UI 依赖，继续使用 Ant Design、Lucide 和现有主题变量。
- 不使用固定高度的执行轮次容器，统一由详情主体滚动。
- 节点输出默认显示紧凑预览，可原位展开查看完整内容。

未采用的方案：

- 页签式详情：视觉更简洁，但会增加导航成本，也会让跨区对照变慢。
- 独立任务详情路由：改动范围大，超出本次布局优化目标。

## 页面结构

1. 弹窗标题区保留任务标题和操作按钮，补充任务类型、运行状态标签。
2. 四个核心问题改为四张独立概览卡，每张卡包含语义图标、标签和值。
3. 原始诉求和任务清单保留双栏，取消固定行高，标题增加对应图标。
4. 执行轮次增加带图标的分区标题、说明和轮次数量。
5. 每个轮次继续按并行或串行布局节点；节点输出改为可展开详情，不再用固定高度硬裁切。
6. 移除执行轮次的内部纵向滚动条，展开内容自然撑开轮次和后续节点。

## 视觉与交互

- 沿用 `--surface-*`、`--border-*`、`--accent-*` 等深浅主题变量。
- 卡片圆角不超过 8px，使用轻量边框、阴影和 1px 位移反馈。
- 人工节点提供清晰的独立操作按钮，避免整卡按钮内嵌可展开控件。
- 移动端四项概览单列、并行节点单列，避免横向挤压。

## 测试与验收

- 样式契约测试验证核心卡片、分区标题、节点输出详情和取消内部纵向滚动。
- 运行前端完整测试和生产构建。
- 浏览器验证深色/浅色主题，以及桌面、平板、手机尺寸下无内容重叠和横向溢出。
