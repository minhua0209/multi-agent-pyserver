# 人工子任务挂起恢复验证用例

## 用例文件

`tests/test_tasks.py`

## 验证目标

验证并行轮次中同时存在 agent 子任务和 human 子任务时：

- agent 子任务可以先自动执行完成。
- human 子任务会挂起等待人工处理。
- 主任务停留在 `running + human_execution`。
- 人工提交结果后，如果本轮所有子任务完成，系统会合并上下文并恢复后续自动流转。

## 覆盖用例

### `test_human_subtask_pauses_round_until_result_is_submitted`

验证步骤：

1. 创建一个处理 agent。
2. Mock 分发 agent，让它在同一轮生成两个子任务：
   - agent 子任务：`Prepare quote`
   - human 子任务：`Approve discount`
3. 人工确认主任务。
4. 验证 agent 子任务完成，human 子任务保持 `running`。
5. 调用 `GET /api/v1/subtasks/human` 查询人工待办。

关键断言：

- 主任务状态为 `running`。
- 主任务节点为 `human_execution`。
- agent 子任务状态为 `succeeded`。
- human 子任务状态为 `running`。
- 人工待办接口能查到挂起的 human 子任务。

### `test_human_subtask_result_resumes_task_flow`

验证步骤：

1. Mock 分发 agent，让它生成一个 human 子任务。
2. 人工确认主任务后，主任务进入 `human_execution`。
3. 调用 `POST /api/v1/subtasks/{subtask_id}/result` 提交人工结果。
4. 验证系统合并人工结果，并继续后续自动流转直到完成。

关键断言：

- 人工提交后主任务状态为 `succeeded`。
- 主任务节点为 `completion_judge`。
- 主任务上下文包含人工输出。
- human 子任务状态变为 `succeeded`。

## 涉及接口

```http
GET /api/v1/subtasks/human
```

查询当前等待人工处理的子任务。

```http
POST /api/v1/subtasks/{subtask_id}/result
```

提交人工子任务处理结果。

示例入参：

```json
{
  "result_status": "succeeded",
  "output": "discount approved",
  "should_complete": true
}
```
